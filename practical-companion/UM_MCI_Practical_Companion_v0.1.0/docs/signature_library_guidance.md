# Signature library guidance

A signature seed in this companion is a **recognizer hint**, not a stable semantic truth. It is intended to help downstream users:
- classify recurring cross-paradigm interactions,
- retrieve analogous cases,
- localize failures,
- bootstrap recognizer learning.

Because signatures are practical and revisable, they belong in the practical companion rather than in the archival core.

## Classification status

A contributed signature seed is a recognition candidate, not a final classification of an edge family. Contributors may state that a pattern appears to instantiate an existing edge family, but maintainer review decides whether it is:

- a useful surface variant of an existing edge family,
- an incorrect classification,
- a seed that should be rewritten, or
- evidence for a future governance proposal.

Validators can check that a signature uses released core IDs and that base/supporting paradigms match the directed edge family. They cannot prove that the methodological classification is substantively correct. Signature reliability therefore depends on curation, evidence, and later review, not on schema validation alone.
