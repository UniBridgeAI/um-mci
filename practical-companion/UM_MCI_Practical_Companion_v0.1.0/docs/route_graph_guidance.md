# Route graphs and stage links

A complex task does not need to be collapsed into a single flat route. In UM-MCI, a **stage** is a local methodological episode with its own base paradigm, dimension commitments, supporting edges, artifact/validation plan, and outcome.

A full task may be represented as a **route graph** over stages. The archival core meta-run schema already supports stage links such as:
- `precedes`
- `feeds`
- `branches_to`
- `merges_into`
- `refines`
- `revises`
- `calls`

This is enough to represent common structures such as chaining, branching, stage-level hierarchy, refinement loops, and simulation-in-the-loop calls.
